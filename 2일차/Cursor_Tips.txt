# Cursor 입문자를 위한 Tips
---
- Cursor는 VS Code 기반 AI 코딩 에디터로, 채팅창에서 코드 작성·수정·설명을 요청할 수 있다.
- @파일명으로 특정 파일을 대화에 첨부하면 AI가 그 내용을 읽고 답한다.
- Agent 모드는 파일 생성·수정·터미널 실행까지 AI가 직접 수행한다.
- Ask 모드는 코드를 읽고 설명만 하며, 파일을 바꾸지 않는다.
- Ctrl+K(인라인)는 커서 위치 코드를 바로 수정하고, Ctrl+L은 채팅창을 연다.
- Rules(.cursor/rules)에 프로젝트 규칙을 적어두면 AI가 매번 그 기준을 따른다.
- AI가 제안한 변경은 diff로 확인한 뒤 Accept/Reject로 승인·거절할 수 있다.
- 민감한 정보(API 키, 비밀번호)는 채팅에 붙이지 말고 .env 등으로 분리한다.
- 요청은 구체적으로("로그인 버튼 색을 파란색으로") 할수록 원하는 결과에 가깝다.
- 잘 안 되면 에러 메시지·파일 경로·원하는 결과를 함께 알려주면 해결이 빨라진다.


# VS Code+Claude와 Cursor의 비교
---
- VS Code+Claude와 비교를 해보면 커서가 토큰을 덜 먹는다.
- LLM채팅창에 이미지 삽입 및 인식도 가능하다.
- Agent 모드에서 여러 파일 수정·생성과 터미널 실행까지 AI가 직접 수행한다 (VS Code+클로드는 주로 조언·부분 수정).
- @Codebase로 프로젝트 전체를 검색·참조할 수 있어 파일을 하나씩 붙이는 수고가 적다.
- Cursor Tab 자동완성이 에디터에 깊게 통합되어 코딩 중 실시간으로 다음 코드를 제안한다.
- 여러 파일 변경을 하나의 작업(Composer/Agent)에서 묶어 검토·적용할 수 있다.
- .cursor/rules에 프로젝트 규칙을 적어두면 매 대화에 자동 반영된다.
- 변경 사항을 diff로 미리 보고 Accept/Reject로 안전하게 적용할 수 있다.
- VS Code 확장은 대부분 호환되지만, Agent·Rules 등 Cursor 전용 기능은 Cursor에서만 쓸 수 있다.
- 구독 하나로 AI 기능이 묶여 있고, VS Code+클로드는 에디터와 AI 구독(API)을 따로 관리해야 한다.
