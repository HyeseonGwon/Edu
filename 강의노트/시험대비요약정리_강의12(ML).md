# 중요 입력 변수를 선택하는 기법들
## 1. 차원 축소
**차원의 저주?**
입력 변수의 수가 너무 많으면 Noise가 발생, 모델이 복잡해지고 **과적합**이 발생 -> Train data에는 잘 맞지만 Test data에는 맞지 않게 된다.

**반도체 데이터의 특징**
의미 없거나 중복되는 입력변수가 존재하는 경우가 많아 차원 축소를 통해 모델을 적합해야 한다.
- FDC에서 하나의 센서값에 대해 mean, max, min, std 등이 여러 변수로 들어가면 '다중공선성' 발생
- 출력 변수와 상관 없는 센서값이 변수로 들어가는 경우
- Large p, small n인 경우가 많다(변수가 많은데 데이터는 적음. 데이터 갯수는 변수 갯수의 10배는 되어야한다)

**어느 단계에서 차원을 축소하나?**
Data scaling 후 train/test 분리하고 나면 train data에서 차원 축소를 시행한다.
#### 차원을 축소하는 방법
1) **mRMR** 
	- minimum-입력 변수들 간의 관계 / Maximum-입출력 변수 간 관계.
	- 변수 수가 많아 빠른 선별이 필요할 때
2) **Boruta**
	- 랜덤으로 만든 가짜 변수보다 중요한 변수만 선택(Random Forest 활용)
	- 선택된 입력변수에는 다중공선성이 일부 남아있지만, 그럼에도 검증 성능은 향상된다.
	- 최종 중요 변수 선별이 필요할 때 쓴다. 변수가 너무 많은 상태에서는 시간이 너무 오래 걸리니 비추

## 2. 모델 해석 방법
#### 예측 모델 해석을 위한 SHAP Value; XAI (Explainable AI)
머신러닝은 회귀계수를 보고 중요한 변수를 알아챌 수 있지만 예측력이 떨어지고,
딥러닝은 예측력이 대체로 높지만 기본적으로 model이 어떤 변수를 중요하게 생각했는지 모른다(black box).
->추정 결과로부터 변수별 기여도를 뽑아내 *추론을 왜 그렇게 했는지 설명*을 해주는 것이 SHAP value이다.

**Shapley Value**
게임 이론으로부터 나온 개념이다. 협업에 참여한 사람들 중 특정 참여자(변수)의 업무 **기여도**가 얼마나 되는지 측정하기 위해 고안되었다.
- 모든 변수들의 부분 집합에 대해 변수i(특정 참여자)가 존재할 때와 존재하지 않을 때 예측 값의 차이를 비교해서 기여도를 평가
- Shapley value = 공정하게 점수 나누는 공식
- SHAP = 그 공식을 AI 예측 설명에 쓴 것
	SHAP으로 구한 숫자 = Shapley value(기여도)

#### 변수의 중요도를 판단하는 두 가지 방법 비교
1) **Random Forest의 feature importances**
	 “이 모델이 트리를 나눌 때 **어떤 변수를 가장 유용하게 썼는가**”  
2) **SHAP Summary Plot**
	 “이 변수가 **각 샘플의 예측을 평균에서 얼마나, 어느 방향으로** 움직였는가”

반도체 공정 데이터로 모델을 해석할 때, RF 중요도는 정상/불량 또는 spec in/out을 가르는 핵심 공정 변수를 빠르게 식별하는 데 유용하다. 다만 개선 방향까지는 알려주지 않는다.  
SHAP은 각 공정 파라미터가 결과값(또는 불량 확률)을 올리는지/내리는지, 얼마나 기여하는지를 보여주므로, 불량 원인 분석이나 특정 target에 맞춘 공정 profile 조정에 더 적합하다.  

>따라서 RF는 1차 변수 선별에, SHAP은 구체적 해석·조치 방향 도출에 활용하는 것이 바람직하다.

## 3. Trace Data를 머신 러닝 입력 데이터로 변환하기
#### FDC data를 변수별로 요약하여 1행으로 만드는 방법
- 장점 : 구현이 간단. 통계량을 사용하면 된다.
- 단점 : 시간에 따른 변화 패턴 정보 사라짐
- 구간별로 나누어 변수를 생성하면 시계열 정보를 남길 수 있다.
- 통계량을 많이 사용하면 변수 수가 배로 늘어남
- 변수가 너무 많으면 **차원의 저주**가 발생할 수 있으므로, 이후 mRMR, Boruta 등의 **차원 축소** 기법을 적용하는 것이 좋음
- 변수별 분포 특성을 분석하고, 저마다 적합한 통계량을 결정할 수도 있다.

# Day 12 Feature Selection Methods — 시험 대비 요약

중요 입력변수를 고르는 기법: **차원 축소(mRMR, Boruta)** + **모델 해석(SHAP/XAI)**

---

## 1. 차원 축소 개요

### 차원의 저주 (Curse of Dimensionality)
- 입력변수(차원)가 과도하면 **Noise↑**, 모델 복잡도↑ → **과적합** (Train↑ / Test↓)
- 고차원일수록 데이터가 공간에서 **희소(Sparsity)** → 학습 느리고 예측 불안정
- $X$간 고상관이면 **다중공선성** → 모형 불안정

### 반도체 데이터 특성
- 의미 없거나 **중복** 변수가 많아 차원 축소가 사실상 필수
- FDC: 센서 1개 → mean/max/min/std 등 다수 변수 → **다중공선성**
- $Y$와 무관한 센서가 섞임
- **Large $p$, Small $n$** (경험칙: 데이터 수 $\gtrsim$ 변수 수의 **10배**)

### 언제 / 무엇을 줄이나
- 순서: **스케일링 → train/test 분할 → train에서 차원 축소** 후 예측 모델 적합  
  (즉 $\hat{Y}=\hat{f}(X)$에서 $X$ 자체를 찾는 단계)
- 장점: 신규 데이터 정확도↑, 학습 속도↑, 시각화 용이

| 구분 | 요지 | 예 |
|------|------|-----|
| **특징 선택 (Feature Selection)** | 중복·무관 변수 **제거**, 중요 원변수 선택 | **mRMR**, **Boruta** |
| **특징 추출 (Feature Extraction)** | 기존 $X$ 조합으로 **잠재변수** 생성 | **PCA** 등 |

### 기법 선택 가이드
| 기법 | 초점 | 언제 |
|------|------|------|
| **mRMR** | min Redundancy + max Relevance | 변수 많아 **빠른 1차 선별** |
| **Boruta** | $Y$와 관련 있는 **모든** 변수 (All Relevant) | **최종 중요 변수** 선별. 변수 폭증 시 시간↑ → 비추 |

## 2. mRMR Algorithm

**mRMR** = **m**inimum **R**edundancy **M**aximum **R**elevance  
- $X$–$X$ 관계 **최소화**, $X$–$Y$ 관계 **최대화**
- 관계 척도: 주로 **Pearson 상관계수** $r(x;y)$  
  - 같이 증감 ≈1 / 반대 ≈−1 / 무관 ≈0  
  - 장점: 계산 간단 / 단점: **선형만**, 이상치 민감  
  - 비선형·이상치 강건: **Chatterjee** 상관계수(2021, 식 생략)

### 알고리즘 요약
1. 후보 $F=\{X_1,\ldots,X_p\}$, 선택집합 $S=\emptyset$, 선택할 개수 $k$(하이퍼파라미터)
2. 각 $r(X_i;Y)$ 계산
3. 첫 변수: $X^*_{\text{first}}=\arg\max_i r(X_i;Y)$ → $S$에 추가
4. $|S|=k$까지 반복:  
   $X^*_{\text{next}}=\arg\max_{i\in F}\left[r(X_i;Y)-\dfrac{1}{|S|}\sum_{j\in S}r(X_i;X_j)\right]$  
   → $Y$와는 크고, **이미 뽑힌 $X_j$와는 작은** 변수 선택 (평균 중복성 차감)

### 예제 요지
- $Y$와 상관 최대 $X_5$ 먼저 → 다음 $r(X_i;Y)-r(X_i;X_5)$로 $X_4$ → 이후 $X_1$ 등  
- 공정 예: 117개→14개($k=14$) 선택 시 다중공선성 완화, $R^2$·MAE 개선

## 3. Boruta Algorithm

- 목표: Minimal Optimal이 아니라 **All Relevant Features** ($Y$ 관련이면 중복 있어도 유지)  
  예: 불량 **혐의 인자 전부** 도출
- 기반: **Random Forest** + 가설검정. 성능 좋으나 **계산 시간↑**

### 3단계
1. **Shadow Feature** 생성  
   - 각 원변수 값을 **무작위 셔플**한 가짜 변수 $S_j$ 추가 → 확장 데이터
2. **Z-score(변수 중요도)** 계산  
   - RF로 원변수·Shadow 중요도 비교  
   - Shadow 중 **최대 Z-score**보다 높으면 해당 원변수에 1(Important), 아니면 0  
   - RF를 $n$번 반복 → 각 변수는 **이항분포** 성공횟수 축적
3. **이항분포 가설검정**으로 선택  
   - $H_0$: 성공확률 $p=0.5$ (불확실) / $H_1$: $p\neq 0.5$  
   - **채택(녹색)**: 성공 많음 → 중요 / **거절(적색)**: 성공 적음 → 비중요 / **불확실(파란)**

### Z-score = Mean Decrease in Accuracy (Permutation Importance)
- OOB에서 성능 측정 → 변수 $j$만 섞은 OOB$_j$로 재측정 → **성능 감소분**을 $T$회 모아 Z-score화  
  $z_j=\bar{z}_j\big/\sqrt{\sum(z_{jt}-\bar{z}_j)^2/(T-1)}$  
- (참고: Mean Decrease in Impurity는 Day10 RF에서 다룸)

### 특징
- 선택 변수에 **다중공선성 일부 잔존** 가능해도 검증 성능은 향상되는 경우 많음
- 공정 예: 117개→10개 선택, 성능 개선

## 4. 모델 해석: SHAP Value (XAI)

### 왜 필요한가
- 단순 모델(선형·나무): **해석↑ / 예측↓**
- 복잡 모델(신경망 등): **예측↑ / Black Box**
- **SHAP**: 추정 결과에서 변수별 기여도를 뽑아 “왜 그렇게 예측했는지” 설명

### 해석 방법 분류
| | **Global** | **Local** |
|--|------------|-----------|
| **Model Specific** | 회귀계수, DT Feature Importance | DT의 개별 Branch |
| **Model Agnostic** | Global Surrogate | **LIME**, **SHAP** |

### Shapley Value (게임이론)
- 협업 참여자(변수)의 **공정 기여도**
- 모든 부분집합에서 변수 $i$ **있을 때−없을 때** 예측 차이(한계기여)를 가중 평균  
  $\phi_i(f,x)=\dfrac{1}{M}\sum_{z'\subseteq x'\setminus i}\binom{M-1}{|z'|}^{-1}\big[f(h_x(z'\cup\{i\}))-f(h_x(z'))\big]$
- **Shapley value** = 기여도 공식 / **SHAP** = 이를 AI 예측 설명에 적용한 것  
  (전체 부분집합 계산은 폭증 → 추정 필요)

### LIME (로컬 대리모형)
- Black box $f$를 지역적으로 흉내 내는 해석 가능 모형 $g$(선형·나무 등)
- 주변 데이터 생성 → 거리에 가중 → $g$ 학습 → 계수로 국소 해석
- 목적: $\xi=\arg\min_g L(f,g,\pi_x)+\Omega(g)$
- 장점: Model-Agnostic, **Local** 직관 / 단점: $\pi_x,\Omega$, 샘플링에 따라 해석 변동·국소 비선형이면 한계

### SHAP = LIME + Shapley 가중
- LIME은 “가까운 점”에 가중 / **Kernel SHAP**은 Shapley식에 맞게 $\pi_x$를 정의해 $\phi_i$가 Shapley가 되도록 함
- 종류: Kernel SHAP(Agnostic, 느림) / Tree·Linear·Deep SHAP(Model-Specific, 권장)

### RF Feature Importance vs SHAP Summary
| | **RF Feature Importance** | **SHAP Summary** |
|--|---------------------------|------------------|
| 질문 | 트리 분할에 **어떤 변수를 자주·유용하게 썼는가** | 각 샘플 예측을 **평균 대비 얼마나·어느 방향으로** 움직였는가 |
| 공정 활용 | Spec in/out·정상/불량 가르는 **핵심 변수 1차 식별** | 파라미터↑/↓가 결과에 미치는 **방향·크기** → 원인분석·공정 조정 |
| 권장 | **1차 선별** | **구체 해석·조치 방향** |

※ DT/RF 중요도는 고카디널리티 범주·연속형 분기가 많아 **과대평가** 경향 → SHAP 선택이 더 안정적인 경우 있음

### 주요 시각화
| Plot | 용도 |
|------|------|
| **Summary (Bar)** | 전역 영향력 순위 |
| **Summary (Scatter)** | 값 크기(색) vs 영향 방향·크기 |
| **Force Plot** | **한 샘플**(또는 시계열 나열) 예측을 올린/내린 변수 |
| **Bar Plot** | 한 샘플의 변수 기여 순위 |
| **Dependence Plot** | 특정 변수 값에 따른 SHAP 변화 |

### SHAP 기반 변수 선택
- **RFE**: Summary에서 영향 최소 변수 제거 반복
- **RFA**: 영향 큰 변수부터 추가, 미미하면 중단
- **Boruta-SHAP**: Boruta의 Z-score 대신 SHAP 사용
- `shap-hypetune`: 변수 선택 + Grid/Random/Bayes 하이퍼옵티 동시 가능

## 5. Trace Data → ML 입력 변환 (요약 통계화)

FDC Trace를 변수별 통계로 **1행(샘플)**으로 요약하는 전처리 (이후 mRMR/Boruta와 연결)

| | 내용 |
|--|------|
| **장점** | 구현 간단 (mean, std, max, min, range 등) |
| **단점** | 시간축 **패턴 정보 소실** |
| 보완 | **구간별** 통계 → 시계열 정보 일부 보존 |
| 주의 | 통계량↑ → 변수 폭증 → **차원의 저주·다중공선성** → **mRMR/Boruta**로 축소 |
| tip | 변수별 분포 보고 적합한 통계량을 다르게 쓸 수 있음 |
