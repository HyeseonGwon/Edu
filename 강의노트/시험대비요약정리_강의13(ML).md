# 제조 데이터 분석을 위한 원인 분석 및 규칙 도출 방법

## 1. 대조 집합 학습을 위한 STUCCO 알고리즘
정상 vs 불량, 또는 고수율 vs 저수율 그룹 간 대비되는 차이를 탐지하고 이를 If-Then 형태의 규칙으로 나타내는 머신러닝 기법이다. 

**대조 집합(Contrast Set, cset)의 예시**
- Rule 1) X1 = A1 -> Gh ; X1열 값이 A1 설비인 경우 고수율 그룹(Gh)에서 자주 등장
- Rule 2-대조 집합) X2 = D2, X3 = G3 -> Gl ; D2, G3 설비를 거친 제품은 저수율 그룹(Gl)에 자주 등장

대조 집합으로 선정되기 위해서는 아래의 두 조건을 만족해야 한다.
조건을 만족하는지는 **STUCCO 알고리즘**으로 검증한다!
1) **Large 조건**
	결과 그룹 간 cset의 support차이(등장 비율)가 최소 $𝛿$만큼 나는가?
	ex) 특정 cset 후보가 불량 data 150개 중 80번, 정상 data 100개 중에 20번 등장한다면 각 support값 차이는 $80/150 - 20/100 = 0.334$가 된다. $𝛿=0.3$이라면 이 cset후보는 Large 조건을 만족한다.
2) **Signiticant 조건**
	 cset 후보와 결과 그룹 간의 독립성을 카이제곱($𝜒2$) 통계량 기반으로 검정한다. cset과 상관 없이 기대되는 불량 확률과 cset을 타고 갔을 때 기대되는 불량 확률이 유의미하게($𝛼$수준에서) 다른지를 본다.
	cf) 가설 검정

*입력 변수도 그룹핑 된 범주형으로 변경해야 STUCCO를 쓸 수 있다.* 대조 집합은 알고리즘이 찾아준다.
모든 조합을 고려하는 것은 오래 걸린다. 유의미하지 않은 노드는 제거하는 것이 효율적인 탐색에 도움이 된다. (가지치기)

**대조 집합 학습(STUCCO)과 의사결정나무의 차이는?**
- 의사결정나무는 다양한 규칙을 생성하기 어렵지만 STUCCO는 조건의 조합을 기준으로 규칙을 찾을 수 있다.
- 특히 Class 불균형이 심할 때는 STUCCO의 성능이 우수할 가능성이 높다. STUCCO는 예측 정확도를 중요시하지 않으므로 Accuracy paradox에 빠질 가능성이 적다. 대신 cset이 두 그룹에서 관측될 차이에 중점을 둔다.
- 출력 변수를 '불량률'과 같은 실수로 할 때에는 회귀나무를 쓴다.

## 2. K-Means Clustering
Y값이 없는 비지도 학습으로, **중심까지의 거리** 기반으로 군집화를 진행한다.

**Clustering의 용도는?
1) 세분화
	- 그룹별 대표 데이터로 전체 데이터를 요약
	- 개별 그룹 내에서의 데이터 특징 분석
2) 클래스 레이블 태깅
	- 지도학습을 위해 군집별로 클래스 레이블을 새롭게 태깅

**Silhouette(실루엣) method : 군집의 개수 k를 설정하는 방법**
- 객체 $i$가 속한 군집 내 data들 간 평균 거리 $a(i)$를 구하고, 다른 군집까지의 평균 거리 $b(i)$보다 얼마나 작은지 확인한다. 
- Sillhouette score $𝑠(𝑖)$는 아래와 같이 구한다. 값이 1에 가까울수록 객체 $i$는 올바른 클러스터에 분류된 것이다.
- $𝑠(𝑖)=(𝑏(𝑖)-𝑎(𝑖)) / max\{𝑎(𝑖),𝑏(𝑖)\},  where −1≤𝑠(𝑖)≤1$
- k를 증가시켜가며 평균 실루엣 값(Silhouette Coefficient)이 최대가 되는 k를 선택한다.

## 3. DBSCAN Clustering (DB-SCAN)
K-Means와 다르게 군집 수를 직접 설정할 필요가 없이 **밀도 기준으로 군집화를 진행**한다. 따라서 이상치 탐지 또한 가능하다. 
- eps의 크기 : 하나의 군집으로 인정할 최대 거리(군집 반지름)
- min_samples 수 : 하나의 군집으로 인정하기 위한 최소 이웃 수
- eps보다 멀리 떨어져 있는 min_samples 개수 이하의 data는 outlier가 된다.

제조 데이터에서 **원인 규칙 도출(STUCCO)** + **비지도 군집(K-Means, DBSCAN)**

---

## 1. 대조집합 학습과 STUCCO

### 개념
- 정상 vs 불량, 고수율($G_H$) vs 저수율($G_L$) 등 **그룹 간 차이**를 찾아 **If-Then 규칙**으로 표현
- **대조집합(Contrast Set, cset)** = 규칙의 **LHS**(조건부). 단일 변수 또는 변수 조합(교호작용)

예) $X_1=A_1 \rightarrow G_H$ / $X_2=D_2,\ X_3=G_3 \rightarrow G_L$

### cset 선정 조건 (STUCCO로 검증)
| 조건 | 내용 |
|------|------|
| **Large** | $\max_{ij}\lvert support(cset,G_i)-support(cset,G_j)\rvert \ge \delta$ |
| **Significant** | 그룹 간 $P(cset\mid G)$가 통계적으로 다름 ($\chi^2$ 독립성 검정) |

- $support(cset,G)=\dfrac{count(cset\cap G)}{count(G)}$
- 예: 불량 80/150, 정상 20/100 → 차이 $0.334$. $\delta=0.3$이면 Large 통과 / $\delta=0.4$면 탈락
- $\delta$↓ → 규칙↑ / $\delta$↑ → 규칙↓ (하이퍼파라미터, try-and-error)

**Significant**: $H_0$: cset⊥그룹(독립) / $H_1$: 독립 아님  
$\chi^2=\sum\dfrac{(O_i-E_i)^2}{E_i}$, $E_i=\dfrac{\text{행합}\times\text{열합}}{\text{전체}}$  
$\alpha=0.05$, df=1일 때 임계치 $\chi^2_{0.05;1}=3.84$ → 초과 시 $H_0$ 기각

### 탐색·전제
- **너비 우선 탐색(BFS)**: LHS 길이 1 → 2 → … (제조에선 보통 길이 2까지)
- 전체 조합은 폭증 → **가지치기**(Effect Size / Statistical / Interest 등)
- 입·출력 모두 **범주형** 필요: 연속 계측값은 범위로 범주화, One-Hot 등 변환
- Event(PM, Recipe·재료 변경 등) 발생 시 설비명을 `A_before_PM` / `A_after_PM`처럼 **분리**해야 함

### STUCCO vs 의사결정나무
| | **STUCCO** | **의사결정나무** |
|--|------------|------------------|
| 규칙 | 조건 조합을 넓게 탐색 | 뿌리부터 경로 중심 → 다양성↓ |
| 입력 | 범주형만 | 연속형 자체 분할 가능 |
| 불균형 | 예측 Accuracy 비중이 낮아 **Accuracy Paradox에 강함** | 다수 클래스에 치우치기 쉬움 |
| 연속 $Y$(불량률) | 고/저 그룹으로 먼저 나눔 | **회귀나무** 사용 |

## 2. K-Means Clustering

- **비지도**: $Y$ 없이 $X$ 유사도(**중심까지 거리**)로 군집
- 목표: 군집 내 동질↑ / 군집 간 이질↑
- 용도: **세분화**(요약·특징 분석), **클래스 라벨 태깅**(이후 지도 학습)

### 알고리즘
1. 임의 $K$개 **Centroid** 선정  
2. 각 점–중심 거리 계산  
3. 최근접 군집 배정 후 중심 갱신  
4. 미배정 점이 있으면 2 반복

### $K$·초기값
- **Silhouette**:  
  $a(i)$=동일 군집 내 평균 거리, $b(i)$=가장 가까운 타 군집까지 평균 거리  
  $s(i)=\dfrac{b(i)-a(i)}{\max\{a(i),b(i)\}},\ -1\le s(i)\le 1$  
  → 평균 Silhouette Coefficient 최대인 $K$ 선택 ($s\approx 1$이면 좋은 배정)
- 한계: $K$ 설정 난이, **초기 중심에 결과 민감** → **K-Means++**(기존 중심에서 먼 점을 다음 중심으로 확률 선택)

---

## 3. DBSCAN

**Density-Based** 군집. $K$를 직접 안 줘도 되고 **Noise=이상치** 탐지 가능

| 하이퍼파라미터 | 의미 |
|----------------|------|
| **eps ($\epsilon$)** | 이웃으로 볼 최대 거리(반지름) |
| **minPts (min_samples)** | 군집 인정 최소 이웃 수 (관례: $\ge$ 변수수+1, 최소 3) |

### 점 유형
- **Core**: eps 안 이웃 $\ge$ minPts  
- **Border**: Core의 군집엔 속하나 자체는 Core 아님  
- **Noise**: 어느 군집에도 안 속함 → **Outlier**

### 장단점
- 장점: $K$ 불필요, 임의 형태 군집, 이상치 대응  
- 단점: **eps에 민감**(작으면 Noise↑, 크면 군집 1개화) / **밀도 불균일** 시 취약

이상치 탐지 계열: 단변량 Box / 다변량 Mahalanobis / **DBSCAN**

---

## 한눈에

```
원인 규칙: STUCCO (Large δ + χ² Significant, 범주형, BFS)
군집: K-Means(거리·K·Silhouette) / DBSCAN(밀도·eps·minPts·Noise)
```
