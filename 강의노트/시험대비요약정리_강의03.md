**1. Transformer의 등장과 핵심 원리**

- **기존 모델(RNN, LSTM)의 한계**: 문장을 순차적으로 처리하여 **병렬 처리**가 어렵고, 문장이 길어질수록 앞부분 정보가 희석되는 문제가 있었음.
- **Transformer (2017)**: "순서대로 보지 말고, 한 번에 보자"는 아이디어를 통해 전체 문장을 동시에 처리함.
- **어텐션(Attention)**: 단어 간의 관계를 계산하여 문맥 이해도를 높이는 기술로, 멀리 떨어진 단어 사이의 관계도 정확히 파악 가능함. "중요하다고 판단된 단어에 더 큰 가중치를 줌"

**2. 프롬프트 엔지니어링 (Prompt Engineering)**

- **개념**: LLM으로부터 더 나은 답변을 얻기 위해 입력 문장(프롬프트)을 설계하는 기술임.
- **In-context Learning**: 모델의 파라미터를 수정하지 않고 프롬프트 내의 **설명, 규칙, 예시**만으로 모델의 행동을 조정하는 방식임.
- **주요 기법**:
    - **Zero-shot**: 예시 없이 지시만 내리는 방식.
    - **Few-shot**: 소수의 예시를 함께 제공하여 답변의 일관성을 높이는 방식.
    - **Chain-of-thought (CoT)**: "단계별로 생각해보자"와 같은 문구를 통해 중간 추론 과정을 생성하도록 유도하는 기법임.

**3. 모델 튜닝 및 학습 단계 (SFT & RLHF)**

- **Supervised Fine-Tuning (SFT)**: 사람이 작성한 '지시-응답' 쌍을 학습시켜 모델이 사용자의 의도를 따르도록 만드는 단계임.
- **RLHF (Reinforcement Learning with Human Feedback)**: SFT 이후, **사람이 선호하는 답변** 방향으로 모델을 미세 조정하는 강화학습 단계임.
- **Fine-tuning vs Prompting**: 프롬프트는 행동을 '유도'하는 수준(Soft constraint)이라면, 파인튜닝은 모델의 기본 행동(Default behavior) 자체를 '고정'하는 것임.

**4. 최신 LLM 트렌드**

- **Scaling Law**: 모델 크기, 데이터 양, 연산량이 커질수록 성능이 향상된다는 법칙임. 단, 학습 데이터의 크기와 **다양성**이 함께 커져야 한다.
- **주요 모델 특징**:
    - **GPT-4/5**: 범용 대화 및 복합 추론 능력이 뛰어남.
    - **Claude**: 긴 문맥 처리 및 전문적인 분석 작업에 강점이 있음.
    - **Gemini**: 구글 서비스와 통합되며 멀티모달(텍스트, 이미지, 음성 등) 지원이 강력함.

-----**[연습 문제: 빈칸 채우기 및 참/거짓]**  
  
**1. 빈칸 채우기**

- (1) Transformer 모델이 기존 RNN의 한계를 극복하고 병렬 처리를 가능하게 만든 핵심 개념은 ( **Attention / 어텐션** )이다.
- (2) 모델을 재학습시키지 않고 입력 문맥만으로 모델의 출력 행동을 변화시키는 현상을 ( **In-context Learning** )이라고 한다.
- (3) RLHF 과정에서 정답을 알려주는 대신 사람이 두 가지 답변 중 더 좋은 것을 선택한 데이터를 기반으로 점수를 예측하는 모델을 ( **Reward Model / 리워드 모델** )이라고 한다.

**2. 참(O) vs 거짓(X) 판단**

- (1) 프롬프트 작성 시 "감사합니다"와 같은 공손한 표현을 많이 쓸수록 답변의 정확도가 비약적으로 상승한다. ( **X** ) - 감정을 배제한 구체적이고 명확한 지시가 더 정확한 답변을 끌어냄.
- (2) 파인튜닝(Fine-tuning)은 모델의 파라미터를 수정하여 모델의 기본 응답 방식을 근본적으로 바꾸는 작업이다. ( **O** ).
- (3) LLM API를 호출할 때 모델의 답변 텍스트는 응답 객체의 `content` 필드에 담겨 있다. ( **O** ).
