# Day 11 Data Preprocessing & Class Imbalance — 시험 대비 요약

온전하지 않거나 모델링에 부적합한 데이터를 **분석에 맞게 보완**하는 방법들을 다룸.  
(결측치·이상치 전처리 + 클래스 불균형 대응 + 하이퍼파라미터 최적화)

---

## 1. 결측치 처리

결측 처리 = **삭제 (Deletion)** 또는 **대치 (Imputation)**

| 방법 | 요지 | 비고 |
|------|------|------|
| **행 삭제** | 변수 내 결측이 **10% 미만**일 때 주로 사용 | 간단·계산량↓ / Sample size↓ |
| **열 삭제** | 결측이 **과도한 변수**는 통째로 제거 | 예: 결측률 90% 센서 |
| **대치** | 결측을 다른 값으로 채움 | 잘못된 값으로 **분포 왜곡** 가능 |

> **실무 Tip**: 변수별 결측률 확인 → 결측률 높은 변수 제거 → 나머지 결측은 대치

### 단일 대치 vs 다중 대치
- **단일 대치 (Single Imputation)**: 결측을 **한 값**으로 보완. 추정 불확실성에 과도한 신뢰 위험
- **다중 대치 (Multiple Imputation)**: 결측 처리된 **$N$개** 데이터셋 생성 후 종합(평균·최빈). 신뢰↑이나 계산 과다 → 강의·현업에선 거의 미룸

### 단일 대치 방법들

#### 0) 값 기반 대치
| 의도 | 예시 값 |
|------|---------|
| 결측 = “없음” | **0** |
| 결측을 **신호**로 활용 | **9999** (큰 값) |
| 결측을 **범주**로 취급 | **Unknown** |

#### 1) 평균화 기법
- 계산 범위: **전체** 또는 **같은 출력변수 $Y$** 그룹별
- 연속형 → **Mean** / 이상치 많으면 → **Median** / 범주형 → **Mode**(최빈값)

#### 2) 확률적 기법
- 범주형: 등장 빈도(전체 또는 동일 $Y$)로 확률판을 만들어 대치
- 불확실 가정 → **현업·반도체에선 잘 안 씀**

#### 3) KNN Imputer
- 결측 행의 **$k$개 최근접 이웃** 값의 평균(또는 거리 가중평균)으로 대치 ($k$=하이퍼파라미터)
- 일반 유클리드 대신 **NaN Euclidean Distance** 사용  
  $\text{NaN Euclidean}(x,y)=w\cdot\text{Euclidean}(x',y')^2$  
  - $x',y'$: NaN 변수를 제거한 벡터  
  - $w=\dfrac{|x|\text{(원래 변수 수)}}{|x'|\text{(공통 비결측 변수 수)}}$  
  - “결측 없는 $x'$차원 거리 → 원래 $x$차원으로 **증폭**”
- 거리 계산 시 **$Y$는 미사용**(입력변수만)

#### 4) 시계열 데이터 대치
| 방법 | 요지 |
|------|------|
| **Forward Fill** | 이전 시점 값으로 대치 |
| **Backward Fill** | 다음 시점 값으로 대치 |
| **linear** | 앞·뒤 값으로 선형 보간 |
| **time** | **시간 간격**을 반영한 선형 보간 (간격 불균등 시) |

## 2. 이상치 탐지

이상치가 있으면 잘못된 가중치·분포 왜곡 우려 → **탐지**가 전처리의 핵심

### 단변량: Box Plot
- **Five Number Summary** + **IQR** $=Q_3-Q_1$
  - $Q_1$(25%), $Q_2$=Median(50%), $Q_3$(75%)
  - Whisker 최소: $Q_1-1.5\,\text{IQR}$ / 최대: $Q_3+1.5\,\text{IQR}$
- Whisker 밖 → **이상치**로 판정 (점으로 표시)
- “한 열(변수)” 내 튀는 값 탐색·설비 간 분포 비교에 유용

### 다변량: Mahalanobis Distance
- 유클리드: $d(X_q,X_i)=(X_q-X_i)^T(X_q-X_i)$
- **마할라노비스**: $d_M(X_q,X_i)=(X_q-X_i)^T S^{-1}(X_q-X_i)$ ($S$=공분산 행렬)
- **밀도 반영**: 밀도 높은 곳 → 조금만 벗어나도 멀게 / 밀도 낮은 곳 → 멀리 있어도 가깝게
- $d^2$는 자유도 $p-1$인 **$\chi^2$ 분포** → 유의수준 $\alpha$(보통 **0.05**)로 이상치 기준 설정

## 3. Class Imbalance 개요

- **Class Imbalance**: 정상(다수) ≫ 불량(소수)
- 실제 난이도는 **중첩도(Overlapping)**가 결정

| Case | 상황 | 대응 |
|------|------|------|
| **1** | 두 클래스 분리 잘 됨 | 일반 ML로 충분 |
| **2** | 일부 중첩 + 양적 불균형 | 분류선이 정상 쪽으로 치우침 → **2종 오류↑**. Resampling / One-Class |
| **3** | 완전 중첩 (변수 부적절) | **신규 입력변수** 필요 (Data-Centric) |

- **Accuracy Paradox**: 전부 “정상” 예측해도 Accuracy 높음 / **Recall↓·2종 오류↑**
- 불균형 해소 시 보통 **2종↓ ↔ 1종↑ Trade-off**. 1종이 치명적이면 비추천
- 해결 축 2가지: **Resampling** / **One-Class Model**

## 4. Under / Over Sampling (Resampling)

클래스 양을 맞춘 뒤 기존 분류기를 적용하는 전처리

| 구분 | 요지 |
|------|------|
| **Undersampling** | 다수 클래스 축소 |
| **Oversampling** | 소수 클래스 증대 |
| Random Under/Over | 단순하나 **편향·과적합** 위험 → 비권장 경향 |

### Tomek Links (Under)
- **Tomek Link**: 서로 다른 클래스이며 **상호 최근접**인 샘플 쌍
- 쌍의 **다수 클래스**를 제거 반복 → 경계 정리, Case 1에 가깝게

### SMOTE (Over)
- Random Over의 단순 복제 과적합 완화
- 소수 샘플 $x_i$의 $k$-NN 중 이웃 $\hat{x}_i$ 선택 후  
  $x_{\text{new}}=x_i+\delta(\hat{x}_i-x_i),\quad \delta\sim U[0,1]$  
  → **가상(합성)** 소수 데이터 생성

### SMOTE + Tomek Links
- SMOTE로 Over 후 Tomek으로 Under → **복합 Resampling**

### ADASYN (Adaptive Synthetic Oversampling)
- SMOTE 개선: 경계(다수 클래스와 인접)에 **더 많은** 합성 샘플 생성
- 불균형도 $d=m_s/m_l$, 생성량 $G=(m_l-m_s)\times\beta$
- $r_i=\Delta_i/K$ (이웃 중 다수 비율) → 정규화 $\hat{r}_i$ → $g_i=\hat{r}_i\times G$개 생성
- 주변에 다수 없으면 $r_i=0$ → 해당 지점 생성 없음

### Sampling + Ensemble
- Sampling마다 경계가 달라짐 → Over/Under를 **여러 번** 반복해 베이스 모델 학습 후 **다수결**로 결합 → 경계 불안정 완화

## 5. One-Class Learning: SVDD

- **정상 데이터만**으로 정상 영역 Boundary 학습 → 영역 밖 = **이상치(≈불량)**
- 극심한 불균형·불량 거의 없을 때 유용. 불량은 **검증**에만 사용
- 대표: **SVDD**, One-Class SVM (강의는 SVDD 중심)

### SVDD 개념·모형
- 정상 영역을 **초구(Hypersphere)**로: 중심 $\boldsymbol{a}$, 반지름 $R$
- 최소 체적: Minimize $R^2$ s.t. $\|\boldsymbol{x}_i-\boldsymbol{a}\|^2\le R^2$
- 오분류 일부 허용:  
  Minimize $f=R^2+C\sum\xi_i$  
  s.t. $\|\boldsymbol{x}_i-\boldsymbol{a}\|^2\le R^2+\xi_i,\ \xi_i\ge 0$
- **$C$**: 오분류 패널티 하이퍼파라미터  
  - $C$ 큼 → 오분류 거의 불허 → 체적↑ → **과적합**  
  - $C$ 작음 → 체적↓ → **과소적합**
- Boundary 위 점 = **Support Vector**

### 비선형 SVDD
- **Kernel**로 고차원에서 초구 학습 → 원래 공간에선 비선형 Boundary
- 고차원 중심까지 거리를 **관리도(Control Chart)**에 올려 SPC식으로 FDC 가능
- 과적합 위험 큼 → 실무에선 신중

## 6. Hyperparameter Optimization

학습 파라미터가 아닌 **하이퍼파라미터**를 탐색해 검증 성능 최적 조합을 찾는 과정

| 방법 | 요지 |
|------|------|
| **Manual Search** | 경험·직관으로 조정 |
| **Grid Search** | 후보 그리드의 **모든 조합** 평가. 간단하나 차원↑ 시 폭발 |
| **Random Search** | 범위 내 무작위 추출. Grid보다 시간대비 성능 우수 평가 많음 |
| **Bayesian Optimization** | (언급만) |

### Grid Search 필요 요소 3가지
1. **하이퍼파라미터 집합** (이산 그리드)
2. **평가 지표** (회귀: $R^2$, MSE / 분류: F1, G-mean 등)
3. **데이터 분할** (검증셋; $k$겹은 객관적이나 Grid와 결합 시 **연산 폭증**)

### 절차
학습/검증/테스트 분할 → 조합마다 학습·검증 평가 반복 → **최적 조합 선정** → 테스트로 최종 평가  
※ 하이퍼파라미터에 성능이 크게 변하므로 **최적화 필수**
