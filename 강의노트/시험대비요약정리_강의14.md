## 1. 여러 Task를 하나의 Agent로 만드는 방법
**좋은 Tool Wrapping이란?**
1) **단일 책임** : 하나의 Tool은 하나의 모델만 담당하게
2) **표준 스키마** : 모든 Tool이 동일한 입출력 형식(JSON) 준수
3) **근거 포함** : 예측값과 함께 **SHAP** 근거 반환
	"설명 가능성을 함께 제시해야 함"
4) **실패 명시** : 분포 밖 입력시 명확한 실패 신호
	"모르면 모른다고 말하게 한다."


## 2. 멀티 모달, VLM (Vision Language Model)
**왜 필요한가?** 
	**고정된 클래스**는 재학습 없이는 새로운 불량(label이 없는 것)에 대응하지 못한다.
	단일 모달리티의 한계-이미지를 보며 "왜"를 설명할 수 없고, 텍스트를 보며 이미지로 표현할 수는 없다.
**Relative Relationship Form**
	세상에는 label이 없는 정보가 절대 다수 존재한다. VLM은 전통적 hard label이 아닌 "이 둘이 더 유사하다"는 상대적인 정보만 사용한다. 단순한 수치나 범주가 아니라 다양한 모드(mode)로 존재할 수 있다. 
	ex) 이미지와 문장의 관계, 상대적인 순서

#### 핵심 용어
**모달리티(Modality)**: 같은 대상을 가리키는 **다른 형태**의 정보(이미지·텍스트·센서 등)
	mode(최빈값)에서 나온 단어. 그냥 데이터 셋(data set)이라고 받아들여도 무방. 

**공동 임베딩 공간**: 서로 다른 모달을 **같은 벡터 공간**으로 번역 → 유사도 비교
	이미지는 굉장히 고차원을 갖는 data인데, 단순 수치 data는 차원이 낮아 차원 수로만 비교하면 묻히는 정보가 될 수 있다. 
	서로 다른 형태의 정보를 함께 다루기 위해, "임베딩"이라는 하나의 공용어를 쓰도록 한다. 이때 공동 임베딩 공간은 번역기와도 같다.
	ex) 텍스트로 이미지 검색, 이미지→자연어 설명, 위치·시점 특정 등

**VLM**: Vision + Language. “다르게 보임”을 넘어 **왜 이상한지 설명**하는 이상탐지로 확장
	예: **Logical AD**(존재·수량·배열 등 논리 제약 위반), LogicQA 등

## 3. CLIP: 자연어로 이미지를 이해하는 모델
Contrastive Language–Image Pre-training  
공동 임베딩 공간에서 유사도 계산으로 학습한다.
대규모 **이미지–텍스트 쌍**으로 **Contrastive Learning**(맞는 쌍↑ / 틀린 쌍↓)한 VLM. 
CLIP이 등장하고 나서 멀티 모달이 잘 작동하기 시작했다.

| 단계  | 내용                                                        |
| --- | --------------------------------------------------------- |
| 학습  | 이미지 인코더 + 텍스트 인코더 → **공동 임베딩**                            |
| 추론  | 클래스 문구를 텍스트로 임베딩 → 이미지와 **유사도 최대** 클래스 선택 (**Zero-shot**) |
**재학습 없이 새로운 분류가 가능한 이유?** 새 불량 시 **프롬프트 한 줄 추가**로 대응 가능
	CLIP은 자연어에 대한 폭넓은 사전 지식을 갖고 있다. 이미지가 입력되면 여러 텍스트 프롬프트들에 대해 유사도를 계산, 최대 유사도를 갖는 텍스트(클래스)를 선택해 최종 예측을 한다.
**왜 CLIP이 실무에서 중요한가?**
	재학습 사이클(데이터 수집->라벨링->학습->배포)을 프롬프트 수정 한 번으로 단축한다: 새 불량 유형 등장 시 프롬프트 목록에 한 줄만 추가하면 대응 가능
**CLIP의 한계**
	미세 결함 분류는 어려워한다... 일반 객체 구분에 최적화 되어있다.


## 4. WinCLIP: 이상 탐지를 위해 학습된 CLIP
사전 정의된 Normal에 해당하는 Text들과 Anomaly에 해당하는 Text들과의 유사도를 계산하는 방법.
주어진 사물의 **상태를 분류**하기 위한 라벨링이 진행된다: 상태를 명확히 하기 위해 '라벨별'로 '상태 단어'와 '텍스트 템플릿'을 이용해 여러 사전 텍스트 프롬프트를 생성한다. 이로부터 나온 임베딩을 평균함.
- 멀티 클래스 AD(Anomaly Detection): 불량이 있다/없다 구분에 관심
- WinCLIP : 불량에 대한 설명에 관심

### Two-class Anomaly Scoring
- **Normal 텍스트** vs **Anomaly 텍스트**와 이미지 유사도 → Softmax → 이상 점수
- 목표: “불량 있다/없다”뿐 아니라 **상태·설명**에 관심

### Compositional Prompt Ensemble (CPE)
- 상태 단어 + 텍스트 템플릿으로 **다수 프롬프트** 생성 → 라벨별 임베딩 **평균** → 대표 Normal/Anomaly 벡터
- 단일 문구보다 표현 조합으로 정확도↑

### Zero-shot Anomaly Segmentation
- **Sliding Window**로 로컬 visual feature 추출
- Window별 이상점수 → 픽셀에 할당, 겹침은 **조화평균**으로 통합
- **Small / Mid / Image** 다중 스케일 수행

### WinCLIP+
- 정상 **reference 이미지 $k$장**(few-shot) Feature 저장
- Query와 최대 유사도 기반으로 점수 보강 → SPADE·PaDiM·PatchCore 대비 양호한 성능 보고

| | Zero-shot WinCLIP | WinCLIP+ |
|--|-------------------|----------|
| 정상 견본 | 불필요(텍스트만) | $k$장 정상 참조 |

## 5. Agent 적용 방향 (요지)

- 제조 **Closed-Loop 디지털 트윈** 등과 연결: 센서 ML Tool + 카메라 **VLM/WinCLIP** Tool을 Agent가 호출
- 예: 설비 카메라 이상 판단, 품질검사 사진 분류 — 예측·SHAP·실패 신호를 표준 JSON으로 반환

---

## 한눈에

```
Model Zoo → Tool Wrapping(표준·SHAP·실패명시) → Agent
고정 클래스 한계 → VLM/CLIP(프롬프트 Zero-shot)
미세·국소 이상 → WinCLIP(CPE + Window 분할, +정상샷)
```
