## 1. 머신러닝 기본 개념

### 머신러닝이란
- 데이터 속 **패턴**을 찾아 함수·규칙·그래프 등으로 표현하는 분야임
- 일반 공학: $Y=f(X)$가 주어짐 / 머신러닝: 데이터 $(X,Y)$로부터 **$f$**를 찾음
- 현실에선 **모집단**을 모르므로 **학습 데이터(표본)**로 $\hat{f}$를 추론함 (추정값에 **hat(^)** 표기)

| 구분 | 기호 | 동의어 |
|------|------|--------|
| 입력변수 | $X$ | 독립변수, 예측변수(**Predictor**), **Feature** |
| 출력변수 | $Y$ | 종속변수, 반응변수(**Response**) |

### 학습 종류
- **지도 학습 (Supervised)**: $Y$ 존재 → $X$–$Y$ 관계 $f$ 추정
	 - **회귀 (Regression)**: 연속형 $Y$ 예측 (예: 두께)
	 - **분류 (Classification)**: 범주형 $Y$(Class) 예측 (예: 불량 여부)
- **비지도 학습 (Unsupervised)**: $Y$ 없음 → $X$ 간 관계에 초점
	- **군집 분석 (Clustering)**: 유사 데이터 그룹핑
	- **주성분 분석 (PCA)**: 차원 축소
	- **연관 규칙 학습**: 거래(웨이퍼 생산 경로)와 수율 연관 도출

### 모델 학습 원리
- 실제 $Y$와 예측 $\hat{Y}=\hat{f}(X)$의 **차이 최소화** 방향으로 학습
- 회귀: **총 오차²** 최소화 / 분류: **총 오분류** 최소화
- 선형회귀 예: $\hat{f}(X)=\hat{\beta}_0+\hat{\beta}_1 X$ — 절편·기울기를 오차 최소화로 추정

## 2. 반도체 데이터 4대 주의 사항

| 문제 | 원인 | 결과 | 대처 |
|------|------|------|------|
| **과적합 (Overfitting)** | $Y$와 무관한 $X$ 다수 | 학습 오차↓, 신규 예측력↓ | $Y$와 상관↑·$X$간 상관↓인 **중요 변수 선택** |
| **다중공선성 (Multicollinearity)** | $X$간 강한 상관 | **회귀계수 불안정**, 예측력↓ | 상관 높은 변수 제거 |
| **클래스 불균형 (Class Imbalance)** | 소수 클래스(불량) 극소 | 다수 클래스에 편향, 소수 예측력↓ | **Resampling**(삭제/증강), **One-Class** 모델 |
| **프로세스 드리프트 (Process Drift)** | 부품 노후화로 입·출력 서서히 변화 | 초기 모델 성능 하락 | **윈도우 학습**으로 주기적 재학습 |

### 과적합 vs 과소적합
- **Overfitting**: 복잡한 모형 → 학습 성능↑, 검증 성능↓
- **Underfitting**: 단순한 모형 → 학습·검증 모두 성능↓
- 적당한 적합 모델 선택이 필요함

### 다중공선성 핵심
- $X$간 **중복성·고상관** → 학습 데이터 변화에 따라 **회귀계수 변동** 심함
- **상관계수 행렬/Heatmap**으로 파악 (예: $|r|>0.9$ 변수 순차 제거)

### 클래스 불균형·Accuracy Paradox
- 정상 99%일 때 전부 “정상” 예측해도 Accuracy 99% → **Accuracy Paradox**
- 소수 불량 미탐지(**2종 오류**, alarm missing)가 더 심각함 / 가성알람은 **1종 오류**

### 이상적 데이터 vs 제조 데이터
- 이상: **Tall and Skinny** = **Small $p$, Large $n$** + $X$간 상관↓ + $X$–$Y$ 상관↑ + Class Balance + Drift 없음
- 제조: **Short and Fat** = **Large $p$, Small $n$** + 과적합·다중공선성·불균형·Drift 혼재

### Data-Centric vs Model-Centric AI
- **Model-Centric**: 모델 구조·최신 알고리즘 개선
- **Data-Centric**: 데이터 품질 고도화 (변수 도입, 증강, Relabeling)
- Andrew Ng 철강 결함 실험: Model-Centric **0%** / Data-Centric **16.9%** 개선
- 예: PM 전후를 `BeforePM_A` / `AfterPM_A`로 분리, 월별 **Dummy 변수**, 드리프트엔 **제품 번호** 사용 (절대 시각은 비권장)

### 학습 파라미터 vs 하이퍼 파라미터
- **학습 파라미터**: 알고리즘이 자동 추정 (예: $\hat{\beta}_0, \hat{\beta}_1$)
- **하이퍼 파라미터**: 사용자가 설정 (예: Ridge의 $\lambda$, $k$겹의 $k$, 윈도우 크기)

## 3. 머신러닝 프로세스

**순서**: 과제 범위 선정 → 고품질 데이터 수집 → 전처리 → 데이터 분할 → 중요 변수 선택/증강 → 모델 학습 → 성능 검증 → 실전 배치

### 고품질 데이터 (Garbage in, Garbage out)
1. Small $p$, Large $n$
2. 이상치 제거·결측치 보완
3. **대표성** 있는 데이터 (Data Drift 반영)
4. **Class Label 일관성**

### 변수 스케일링
| 방법          | 특징                                           |
| ----------- | -------------------------------------------- |
| **Z-Score** | $Z=(X-\bar{X})/S_X$ → 평균 0, 표준편차 1. 일반적으로 선호 |
| **Min-Max** | $[0,1]$ 정규화. **이상치** 있으면 부적합                 |

**스케일링 필요**: K-NN, SVM, 인공신경망, PCA (+ Lasso/Ridge)  
**불필요**: 선형/로지스틱 회귀, 의사결정나무, Bagging/Boosting  
※ 거리·분산 기반 알고리즘은 스케일 영향이 큼

### 데이터 분할
- 일반 비율 **학습:검증:테스트 = 5:3:2**
  - 학습: $\hat{f}$ 추정 / 검증: 후보 모델 비교·과적합 판단 / 테스트: **최종 성능**만 평가
- **$k$겹 교차검증**: 학습 데이터를 $k$개로 나눠 검증 평균 (주로 $k=5$ 또는 $10$). 최종 모델은 전체 학습 데이터로 재학습 후 테스트
- **윈도우 교차검증** (드리프트 대응): 랜덤 분할로는 Drift 검증 불가
  - **확장 윈도우**: 학습 구간을 시간에 따라 확장
  - **슬라이딩 윈도우**: 윈도우 크기 고정, 과거 데이터 제거하며 이동

## 4. 성능 지표

### 회귀
| 지표 | 요지 |
|------|------|
| **MAE** | 절대오차 평균. 단위 직관적 |
| **MAPE** | 상대(%) 오차. 단위 몰라도 비교 용이. $y_i\neq 0$ |
| **MSE** | 오차 제곱 평균. **큰 오차에 민감**. 학습 목적에 자주 사용 |
| **RMSE** | $\sqrt{\text{MSE}}$. 원단위 환산 |

### 분류 — 혼동 행렬
- 관심 사건(보통 **불량/이상**)을 **Positive**로 설정
- **TP / TN**: 맞춤 / **FP**: 정상→불량(1종) / **FN**: 불량→정상(2종)

| 지표 | 식·의미 |
|------|---------|
| **Accuracy** | $(TN+TP)/\text{전체}$. 불균형 시 부적절 |
| **Recall (재현율)** | $TP/(FN+TP)$. 불량 탐지율 = $1-\beta$(**2종 오류**) |
| **Precision (정밀도)** | $TP/(FP+TP)$. 이상 예측의 신뢰도 |
| **Specificity (특이도)** | $TN/(TN+FP)$. 정상 정확도 = $1-\alpha$(**1종 오류**) |
| **F1 Score** | Precision·Recall의 **조화평균** |
| **G-Mean** | $\sqrt{\text{Specificity}\times\text{Recall}}$ |
| **AUC** | ROC 곡선 아래 면적 |

※ 클래스 불균형 제조 데이터에선 Accuracy보다 **F1 / G-Mean / Recall**이 중요함

## 5. 반도체 데이터 6대 분석 난제

1. **Complex Data Types** — Sensor Trace, Image 등
2. **High Dimensional Problem** — Large $p$, Small $n$
3. **Class Imbalance** — 정상 ≫ 불량
4. **Open Set Problem** — **미지의 신규 불량** 출현
5. **Incorrect Class Labeling** — 사람 라벨 오류 → GIGO
6. **Process Drift** — 소모성 부품 노후화

### 참(O) / 거짓(X)

1. 비지도 학습의 PCA는 출력변수 $Y$를 이용해 차원을 축소함.  
   **X** (PCA는 $Y$ 없이 입력변수 $X$의 차원을 축소함)

2. 회귀 학습은 총 오분류를 최소화하는 방향으로 진행함.  
   **X** (회귀는 총 오차² 등 오차 지표 최소화 / 오분류는 분류)

3. 과적합 시 복잡한 모형은 학습 성능은 높고 검증 성능은 낮아짐.  
   **O**

4. 다중공선성이 있으면 학습 데이터가 바뀌어도 회귀계수는 거의 변하지 않음.  
   **X** (반대로 회귀계수 변동성이 커짐)

5. Accuracy만으로 불량 탐지 성능을 평가해도 클래스 불균형에서 충분함.  
   **X** (Accuracy Paradox로 다수 클래스에 편향될 수 있음)

6. 프로세스 드리프트 대응을 위해 랜덤 $k$겹 교차검증만으로도 충분함.  
   **X** (시간에 따른 분포 변화를 보려면 윈도우 교차검증이 필요함)

7. Min-Max 스케일링은 이상치(Outlier)가 있을 때 특히 적합함.  
   **X** (이상치에 민감하여 부적합. Z-Score를 더 선호)

   **O**

9. 테스트 데이터는 모델 선택(후보 비교)에 사용하고, 검증 데이터로 최종 성능만 봄.  
   **X** (검증으로 모델 선택·비교, 테스트로 최종 성능 평가 — 역할이 반대)

10. Data-Centric AI는 모델 구조 변경보다 데이터 품질·라벨·변수 설계 개선에 초점임.  
    **O**

11. FN(불량을 정상으로 예측)은 1종 오류에 해당함.  
    **X** (FN은 2종 오류 $\beta$ / FP가 1종 오류 $\alpha$)

12. 제조 현장에서는 Accuracy보다 F1 Score나 G-Mean을 더 자주 씀.  
    **O**
