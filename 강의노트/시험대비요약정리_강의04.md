**1. LLM의 구조적 한계와 환각(Hallucination)**

- **지식의 고착화**: 학습 데이터 기반의 ==**고정 지식(Static Knowledge)**== 만을 가지며, 학습이 멈춘 시점 이후의 정보(**Training Cutoff**)나 사내 내부 데이터는 알지 못함.
- **환각(Hallucination)**: LLM은 정답 DB가 아니라 **다음 단어를 확률적으로 예측**하는 엔진이므로, 모르는 내용도 그럴듯하게 지어낼 수 있음.
- **Fine-tuning의 한계**: 행동 방식 교정에는 효과적이나 비용이 많이 들고 지식의 지속 업데이트가 어려우며, 근거를 확인하는 **추적성(Traceability)** 이 부족함. 또한 새 데이터에 대해 기존 능력이 저하될 우려도 있다.

>그래서 Tool calling과 RAG가 필요하다.

**2. 도구 호출(Tool Calling)**

- 종류: 검색, 계산, DB조회, 문서처리, RAG 등...
- **정의**: LLM이 스스로 판단하여 외부 도구(함수)를 호출하는 기능으로, LLM에게 **실제 행동 능력(손발)** 을 달아주는 방법임.
- **구성 요소**: LLM이 도구를 식별하는 **이름(Name)**, 사용 여부를 판단하는 **설명(Description)**, 데이터 형식을 정의하는 **입출력 스키마(Schema)** 가 필요함.
- **장점**: 실시간 데이터 접근이 가능해지며, 정확한 수치 계산이나 DB 조회를 통해 **환각 위험을 최소화**함.

**3. RAG(Retrieval-Augmented Generation)의 개념 및 원리**
==많은 Hallucination이 Generation이 아니라 Retrieval 실패에서 시작됨==

- **개념**: **검색으로 보강된 생성** 방식으로, 질문과 관련된 문서를 외부 지식 소스에서 먼저 찾은 뒤 그 내용을 참고하여 답변을 생성함.
- **장점**: 최신 정보 반영이 쉽고, 답변의 **근거(출처) 제시**가 가능하여 신뢰성이 높음.
- **의미 기반 검색(Semantic Search)**: 키워드가 일치하지 않아도 문장의 의미를 **벡터(Embedding)** 로 변환하여 유사도를 기반으로 관련 정보를 찾아냄.

**4. RAG의 동작 프로세스**

- **Offline Process**: 문서 수집 → 전처리 → **청킹(Chunking, 의미 단위 분할)** → 임베딩 → **벡터 DB 저장** 순으로 진행됨.
- **Online Process**: 질문 입력 → **리트리버(Retriever, 관련 문서 검색)** → **리랭커(Reranker, 정밀 재정렬)** → 프롬프트 구성 → LLM 생성 순으로 진행됨.

-----**[연습 문제: 빈칸 채우기 및 참/거짓]**  
  
**1. 빈칸 채우기**

- (1) LLM이 특정 시점 이후의 사건이나 정보를 알지 못하는 한계를 ( **학습 단절 / Training Cutoff** )(이)라고 한다.
- (2) 검색된 문서들 중 질문과 실제 가장 유용한 문서를 정밀하게 다시 평가하여 순위를 매기는 단계를 ( **리랭커 / Reranker** )(이)라고 한다.
- (3) 단어와 문장을 숫자 벡터로 변환하여 의미적 유사도를 계산할 수 있게 만드는 기술은 ( **임베딩 / Embedding** )이다.

**2. 참(O) vs 거짓(X) 판단**

- (1) RAG 시스템에서 환각(Hallucination)이 발생하는 가장 큰 원인 중 하나는 검색(Retrieval) 단계의 실패이다. ( **O** ).
- (2) 도구 호출(Tool Calling) 시 도구의 이름(Name)보다 설명(Description)이 LLM의 판단에 더 중요한 역할을 한다. ( **O** ).
- (3) 벡터 공간에서 의미적으로 가깝다고 해서 그것이 반드시 질문에 대한 정확한 정답임을 보장하지는 않는다. ( **O** ).
