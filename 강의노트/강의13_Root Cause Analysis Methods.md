# 제조 데이터 분석을 위한 원인 분석 및 규칙 도출 방법

## 1. 대조 집합 학습을 위한 STUCCO 알고리즘
정상 vs 불량, 또는 고수율 vs 저수율 그룹 간 대비되는 차이를 탐지하고 이를 If-Then 형태의 규칙으로 나타내는 머신러닝 기법이다. 

**대조 집합(Contrast Set, cset)의 예시
- Rule 1) X1 = A1 -> Gh ; X1열 값이 A1 설비인 경우 고수율 그룹(Gh)에서 자주 등장
- Rule 2-대조 집합) X2 = D2, X3 = G3 -> Gl ; D2, G3 설비를 거친 제품은 저수율 그룹(Gl)에 자주 등장

대조 집합으로 선정되기 위해서는 아래의 두 조건을 만족해야 한다.
조건을 만족하는지는 **STUCCO 알고리즘**으로 검증한다!
1) **Large 조건**
	결과 그룹 간 cset의 support차이(등장 비율)가 최소 $𝛿$만큼 나는가?
	ex) 특정 cset 후보가 불량 data 150개 중 80번, 정상 data 100개 중에 20번 등장한다면 각 support값 차이는 $80/150 - 20/100 = 0.334$가 된다. $𝛿=0.3$이라면 이 cset후보는 Large 조건을 만족한다.
2) **Signiticant 조건**
	 cset 후보와 결과 그룹 간의 독립성을 카이제곱($𝜒2$) 통계량 기반으로 검정한다. cset과 상관 없이 기대되는 불량 확률과 cset을 타고 갔을 때 기대되는 불량 확률이 유의미하게($𝛼$수준에서) 다른지를 본다.
	cf) 가설 검정

*입력 변수도 그룹핑 된 범주형으로 변경해야 STUCCO를 쓸 수 있다.* 대조 집합은 알고리즘이 찾아준다.
모든 조합을 고려하는 것은 오래 걸린다. 유의미하지 않은 노드는 제거하는 것이 효율적인 탐색에 도움이 된다. (가지치기)

**대조 집합 학습(STUCCO)과 의사결정나무의 차이는?
- 의사결정나무는 다양한 규칙을 생성하기 어렵지만 STUCCO는 조건의 조합을 기준으로 규칙을 찾을 수 있다.
- 특히 Class 불균형이 심할 때는 STUCCO의 성능이 우수할 가능성이 높다. STUCCO는 예측 정확도를 중요시하지 않으므로 Accuracy paradox에 빠질 가능성이 적다. 대신 cset이 두 그룹에서 관측될 차이에 중점을 둔다.
- 출력 변수를 '불량률'과 같은 실수로 할 때에는 회귀나무를 쓴다.

## 2. K-Means Clustering
Y값이 없는 비지도 학습으로, **중심까지의 거리** 기반으로 군집화를 진행한다.

**Clustering의 용도는?
1) 세분화
	- 그룹별 대표 데이터로 전체 데이터를 요약
	- 개별 그룹 내에서의 데이터 특징 분석
2) 클래스 레이블 태깅
	- 지도학습을 위해 군집별로 클래스 레이블을 새롭게 태깅

**Silhouette(실루엣) method : 군집의 개수 k를 설정하는 방법**
- 객체 $i$가 속한 군집 내 data들 간 평균 거리 $a(i)$를 구하고, 다른 군집까지의 평균 거리 $b(i)$보다 얼마나 작은지 확인한다. 
- Sillhouette score $𝑠(𝑖)$는 아래와 같이 구한다. 값이 1에 가까울수록 객체 $i$는 올바른 클러스터에 분류된 것이다.
- $𝑠(𝑖)=(𝑏(𝑖)-𝑎(𝑖)) / max\{𝑎(𝑖),𝑏(𝑖)\},  where −1≤𝑠(𝑖)≤1$
- k를 증가시켜가며 평균 실루엣 값(Silhouette Coefficient)이 최대가 되는 k를 선택한다.

## 3. DBSCAN Clustering (DBS-CAN)
K-Means와 다르게 군집 수를 직접 설정할 필요가 없이 **밀도 기준으로 군집화를 진행**한다. 따라서 이상치 탐지 또한 가능하다. 
- eps의 크기 : 하나의 군집으로 인정할 최대 거리(군집 반지름)
- min_samples 수 : 하나의 군집으로 인정하기 위한 최소 이웃 수
- eps보다 멀리 떨어져 있는 min_samples 개수 이하의 data는 outlier가 된다.
