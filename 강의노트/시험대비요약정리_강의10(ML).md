# Day 10 Advanced Prediction Models — 시험 대비 요약

---

## 1. 앙상블 개요

- **앙상블 (Ensemble)**: 여러 베이스 모델을 학습한 뒤 예측을 **결합**(다수결·평균)해 최종 예측
- 장점: 복잡한 $X$–$Y$ 관계 학습, 개별 오류 **상쇄**, 신뢰성↑ (집단 지성)
- 계열: **Bagging / Boosting / Stacking / Blending**
- 핵심은 **데이터·모델 다양성**. 과적합 쉬운 모델(예: 나무)일수록 앙상블 효과가 큼
- 테이블 데이터는 앙상블이 강세인 경우가 많음

---

## 2. Bagging

- **Bagging** = **Bootstrap Aggregating**
- **붓스트랩 샘플링**: 학습 데이터에서 **복원 추출**로 여러 표본 생성 → 표본마다 베이스 모델 학습
- 표본 크기 = 원본 $n$일 때, 평균적으로 약 **63.2%**만 사용(약 **36.8%**는 OOB, 검증에 활용 가능)
- 표본 내·표본 간 **중복** 존재하나 완전 동일하지는 않음 → **다양성** 확보
- 결합: 분류 → **다수결** / 회귀 → **평균**

---

## 3. Random Forest

- Bagging + 의사결정나무 + **무작위 변수 선택**
- 각 노드에서 입력변수 **일부만 랜덤 선택** 후, 그중 **정보 이득** 최대 변수로 분할  
  (관례: 분류 $\sqrt{p}$, 회귀 $p/3$)
- 붓스트랩 + 변수 랜덤 → 나무 간 다양성↑, 과적합↓
- **변수 중요도**: **Mean Decrease in Impurity**  
  - 노드 $t$에서 $X_i$의 정보 이득 $\Delta_i(t,X_i)$를 구하고, 데이터 비중 $p(t)$로 가중 평균  
  - $Imp(X_i)=\dfrac{1}{N_T}\sum_T\sum_{t: X_i\text{ 사용}} p(t)\cdot\Delta_i(t,X_i)$  
  - 혐의 인자 관리에 유용

---

## 4. Boosting 개요

| 구분 | Bagging | Boosting |
|------|---------|----------|
| 학습 방식 | **병렬** (독립) | **순차** (이전 오류 집중) |
| 초점 | 다양성·분산↓ | 잔차·편향↓ |
| 위험 | 상대적으로 안정 | **과적합** 주의 |

- Adaptive Boosting 계열 / **GBM** 계열 (XGBoost, LightGBM, CatBoost)

---

## 5. GBM (Gradient Boosting Machine)

- $m$라운드 베이스 모델 $h_m$이 이전 결합 $H_{m-1}=h_1+\cdots+h_{m-1}$의 **잔차(예측 오류)**를 학습
- 최종: $H_M(\boldsymbol{x})=H_{M-1}(\boldsymbol{x})+\nu\, h_M(\boldsymbol{x})$
- **Learning Rate** $\nu$ ($0<\nu\le 1$): 베이스 영향력 **수축** → 과적합 방지 하이퍼파라미터
  - $\nu$ 너무 작음 → **과소적합** / 너무 큼 → **과적합**
  - $\nu$와 `n_estimators`는 **Trade-off** → 함께 최적화
- $h_0$: 회귀에선 보통 **$Y$ 평균**. 베이스는 주로 **CART**
- Leaf 출력(GBM): (노드 잔차 합) / (노드 데이터 수)
- 분류도 확률 예측 형태로 회귀적 학습

---

## 6. XGBoost

- GBM + 목적함수에 **트리 규제항** $\Omega(h_m)$
- $\Omega(h_m)=\gamma T+\dfrac{1}{2}\lambda\sum_{j=1}^{T}w_j^2$
  - $T$: 끝 노드 수 / $w_j$: $j$번째 Leaf 출력
  - **$\gamma$**: 복잡도(끝노드) 패널티 → 클수록 **가지치기↑**
  - **$\lambda$**: Leaf 출력 수축 → 클수록 $|w|$↓ (과적합↓)
- Leaf 출력: $w_j=\dfrac{\text{노드 잔차합}}{\text{노드 데이터 수}+\lambda}$  
  - $\lambda=0$이면 GBM과 동일
- $\nu,\gamma,\lambda$ 성능 민감 → **하이퍼파라미터 최적화 필수**

---

## 7. LightGBM

- 목표: GBM 대비 **학습 속도·메모리** 개선 (데이터 많을 때 유리, XGBoost 대비 최대 ~38배 빠름 사례)

| 기법 | 요지 |
|------|------|
| **GOSS** | 잔차 $\|e\|$ 큰 데이터는 전부, 작은 데이터는 Subsample → 잔차 기반 샘플링 |
| **Leaf-Wise** | Loss(잔차²합) 큰 Leaf만 분할. (기존 GBM/XGB는 **Level-Wise** 균형 분할) |
| Exclusive Feature Bundling | (강의 생략) |

---

## 8. CatBoost

- 초점: (1) 학습 과적합 (2) 범주형 Encoding의 **Target Leakage** 완화
- 특히 **범주형 변수 많은** 데이터에 강점
- 핵심 기법: **Ordered Boosting**(생략) / **Ordered Target Encoding**

### 범주형 Encoding 비교
| 방법 | 요지 | 이슈 |
|------|------|------|
| **One-Hot** | 범주→0/1 벡터 | 변수 폭증·과적합 |
| **Target Encoding** | 범주별 $Y$ 평균으로 치환 | **Target Leakage**, 이상치 민감 |
| **Ordered Target Encoding** | 임의 시간순 정렬, $t$시점 Encoding 시 **$t-1$까지**만 사용 | Leakage 방지 |

- Ordered TE:  
  $\text{Category}_{c,t}=\dfrac{n_{c,t}\cdot p_{c,t}+a\cdot p_{global,t}}{n_{c,t}+a}$  
  ($a$: 통제 파라미터, $p_c$와 $p_{global}$ 가중평균 → 이상치 둔감)

---

## 9. Stacking / Blending

| | **Stacking** | **Blending** |
|--|--------------|--------------|
| 구조 | 여러 모델 예측 → **meta-feature** → **meta-model**로 재예측 | 동일 |
| 차이 | meta 학습에 **예측값만** 사용 | 원본 $X$에 **예측값을 결합**해 meta 학습 |

- meta-feature: 1차 모델들의 예측값 / meta-model: 이를 입력으로 쓰는 최종 ML
- 예: SVM + RF + Logistic → meta로 LightGBM → 개별 모델보다 성능↑ 사례

---

## 한눈에 정리

```
앙상블
 ├─ Bagging ── Random Forest (붓스트랩 + 변수 랜덤)
 ├─ Boosting
 │    ├─ GBM (잔차 순차 + ν)
 │    ├─ XGBoost (GBM + γ,λ 규제)
 │    ├─ LightGBM (GOSS + Leaf-Wise, 빠름)
 │    └─ CatBoost (범주형·Ordered TE)
 └─ Stacking / Blending (meta-model)
```
