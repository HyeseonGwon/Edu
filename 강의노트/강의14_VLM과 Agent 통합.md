## 1. 여러 Task를 하나의 Agent로 만드는 방법
**좋은 Tool Wrapping이란?**
1) 단일 책임 : 하나의 Tool은 하나의 모델만 담당하게
2) 표준 스키마 : 모든 Tool이 동일한 입출력 형식(JSON) 준수
3) 근거 포함 : 예측값과 함께 SHAP 근거 반환
	"설명 가능성을 함께 제시해야 함"
4) 실패 명시 : 분포 밖 입력시 명확한 실패 신호
	"모르면 모른다고 말하게 한다."


## 2. 멀티 모달, VLM (Vision Language Model)
**왜 필요한가?** 
	**고정된 클래스**는 재학습 없이는 새로운 불량(label이 없는 것)에 대응하지 못한다.
**Relative Relationship Form**
	세상에는 label이 없는 정보가 절대 다수 존재한다. VLM은 전통적 hard label이 아닌 "이 둘이 더 유사하다"는 상대적인 정보만 사용한다. 단순한 수치나 범주가 아니라 다양한 모드(mode)로 존재할 수 있다. 
	ex) 이미지와 문장의 관계, 상대적인 순서
**모달리티(Modality)**
	mode(최빈값)에서 나온 단어. 그냥 데이터 셋(data set)이라고 받아들여도 무방. 같은 것을 가리키지만 다른 형태의 정보.
**공동 임베딩 공간**
	이미지는 굉장히 고차원을 갖는 data인데, 단순 수치 data는 차원이 낮아 차원 수로만 비교하면 묻히는 정보가 될 수 있다. 
	서로 다른 형태의 정보를 함께 다루기 위해, "임베딩"이라는 하나의 공용어를 쓰도록 한다. 이때 공동 임베딩 공간은 번역기와도 같다.
	ex) 텍스트로 이미지 검색, 이미지로부터 자연어 설명
	ex) 이미지 내 객체 위치 특정, 영상 내 이벤트 시점 특정
> VLM은 "다르게 보인다"를 넘어 "왜 잘못됐는지 설명한다"는 차원으로 이상탐지를 확장한다.

## 3. CLIP: 자연어로 이미지를 이해하는 모델
**"대규모 이미지-텍스트 쌍 데이터"** 를 활용한 멀티 모달 metric learning 기반 모델이다. CLIP이 등장하고 나서 멀티 모달이 잘 작동하기 시작했다.
이미지 인코더(ResNet 등)와 텍스트 인코더(Transformer)로 공동 임베딩 공간에 대규모 이미지-텍스트 쌍을 만들어 유사도를 학습시킨다.
**재학습 없이 새로운 분류가 가능한 이유?**
	CLIP은 자연어에 대한 폭넓은 사전 지식을 갖고 있다. 이미지가 입력되면 여러 텍스트 프롬프트들에 대해 유사도를 계산, 최대 유사도를 갖는 텍스트(클래스)를 선택해 최종 예측을 한다.
**왜 CLIP이 실무에서 중요한가?**
	재학습 사이클(데이터 수집->라벨링->학습->배포)을 프롬프트 수정 한 번으로 단축한다: 새 불량 유형 등장 시 프롬프트 목록에 한 줄만 추가하면 대응 가능
**CLIP의 한계**
	미세 결함 분류는 어려워한다... 일반 객체 구분에 최적화 되어있다.

## 4. WinCLIP: 
사전 정의된 Normal에 해당하는 Text들과 Anomaly에 해당하는 Text들과의 유사도를 계산하는 방법.
주어진 사물의 **상태를 분류**하기 위한 라벨링이 진행된다: 상태를 명확히 하기 위해 '라벨별'로 '상태 단어'와 '텍스트 템플릿'을 이용해 여러 사전 텍스트 프롬프트를 생성한다. 이로부터 나온 임베딩을 평균
- 멀티 클래스 AD(Anomaly Detection): 불량이 있다/없다 구분에 관심
- WinCLIP : 불량에 대한 설명에 관심
